# 架构与执行计划

## 技术选型确认

1. **数据存储格式**：对象序列化 (Object Serialization)。
2. **照片存储**：Base64 编码字符串随对象一起存储。
3. **拼音支持**：引入 `pinyin4j` 依赖进行处理。
4. **自定义列交互**：在表格右键弹出菜单选择显示/隐藏列。
5. **联系人调整分组**：同时支持“右键菜单”与“左侧树状节点鼠标拖拽”两种方式。

## 模块划分与架构设计

1. **Model 层**
   - `Contact`: 包含姓名、电话、手机、IM、邮箱、主页、生日、照片(Base64)、单位、地址、邮编、所属组、备注（需实现 `Serializable`）。
2. **Service / DAO 层**
   - `DataStorageService`: 基于 `ObjectOutputStream`/`ObjectInputStream` 的磁盘读写。
   - `ContactManager`: 联系人列表管理与按拼音/条件检索逻辑。
3. **UI 层**
   - `MainController`: 左侧分组树（支持拖放）、右侧表格（支持右键显隐列及分配分组）。
   - 编辑/导入等对应弹窗控制器。

## 阶段执行步骤

- **阶段一：数据模型升级与持久化**。完成 `Contact` 实体，实现基于对象序列化的持久化存储。
- **阶段二：核心界面与联系人管理**。主界面表格展示所有字段，实现右键显隐列；完成增删改交互。
- **阶段三：联系组管理与分配**。实现新建/删除分组，以及右键和拖拽分配分组功能。
- **阶段四：高级搜索与排序**。引入 `pinyin4j`，完成声母/全拼/姓名/电话模糊搜索，及拼音归类排序。
- **阶段五：导入导出功能**。实现 CSV 和 vCard 格式的数据导入与导出。
